## Design Context

### Users
Power users and developers who need fast, reliable screenshot and screen recording capabilities. They use this tool frequently throughout the day, often in focused work contexts. The interface should feel like a natural extension of their workflow—not an interruption.

### Brand Personality
**Quietly capable, technically refined, effortless**

- **Quietly capable**: Does the job without demanding attention
- **Technically refined**: Precision and polish in every detail
- **Effortless**: Actions feel natural and immediate

The personality draws from:
- Terminal/editor aesthetics (monospace, flat, minimal)
- High-end developer tools (clean, focused, purposeful)
- The concept of a "synapse" - quick, neural, almost invisible in its efficiency

### Aesthetic Direction
**Refined minimalism with subtle neural accents**

The design should feel:
- Like a premium native desktop application
- Immediately familiar to developers and power users
- Restrained in color and motion - letting the functionality speak
- Dark by default (screenshots often taken in focused work sessions)

**Visual approach**:
- Monospace typography throughout (IBM Plex Mono)
- Flat design - no shadows, minimal depth
- Color as subtle accent rather than loud statement
- Motion that serves interaction, never decoration
- The "neural" concept expressed through connection lines and nodes, not neon sci-fi effects

**What this should NOT look like**:
- Gaming aesthetic with neon glows and aggressive animations
- Generic SaaS dashboard with gradient hero sections
- Consumer app with playful illustrations and bouncy interactions

### Design Principles

1. **Restraint over spectacle**: Every visual element must earn its place. If it doesn't communicate state or guide action, remove it.

2. **Quiet confidence**: The interface doesn't need to shout about what it can do. Let the speed and reliability speak.

3. **Technical precision**: Align to grid, consistent spacing, purposeful transitions. The polish signals quality.

4. **Motion serves function**: Animations should clarify state changes and provide feedback. Never decorative flourishes.

5. **The neural metaphor, expressed quietly**: Connection lines, subtle node highlights, smooth signal flow. Think neural pathways, not lightning bolts.

### Color Strategy

- **Neutrals dominate**: 85%+ of the interface
- **Accent appears sparingly**: Only for interactive elements and states
- **No neon, no gradients**: Keep colors grounded in the physical world
- **Tints, not pure colors**: Everything slightly muted for sophistication

### Motion Strategy

- **Easing**: cubic-bezier(0.25, 1, 0.5, 1) for smooth deceleration
- **Durations**: 150-300ms for micro-interactions
- **Distance**: Small movements (2-8px) that feel precise
- **Triggers**: Only on user interaction or meaningful state change

## Recent Refinements (April 2026)

### 1. SynapseLogo - Quiet Neural Animation
**Before**: Constant pulsing rings, traveling electric pulses, multiple simultaneous glow effects, overly complex animations

**After**: 
- No constant animation (pulsing removed entirely)
- Motion only triggers on hover
- Simplified to: subtle glow, line brighten, node pulse
- Smooth cubic-bezier easing (0.25, 1, 0.5, 1)
- Reduced filter effects for better performance
- Letter cascade effect: 40ms stagger per letter on hover

### 2. Update Available - Inline Banner → Modal
**Before**: Inline banner at top of mode selector, cluttered layout

**After**:
- Centered modal presentation
- Clean icon treatment (download icon in subtle accent container)
- Clear action hierarchy: Install Now (primary) / Later (secondary)
- Smooth enter/exit animations (200ms)
- Backdrop blur for focus
- Version info in muted footer text

### 3. Tray Context Menu - New Custom UI
**Before**: Native OS tray menu (limited customization)

**After**:
- Custom UI modal positioned near tray
- Organized sections: Quick Capture, App Options, Quit
- Keyboard shortcuts displayed (⇧⌘4, ⌘, etc.)
- Consistent with app design language
- Smooth enter/exit animations
- Right-click on tray triggers UI (Rust event emission)
